# Verificación de normalización y diseño

## Checklist de verificación

- [ ] Todas las tablas tienen clave primaria definida.
- [ ] No existen atributos multivaluados en las tablas.
- [ ] No existen dependencias parciales respecto a claves compuestas.
- [ ] No existen dependencias transitivas.
- [ ] Las claves foráneas están correctamente declaradas.
- [ ] Los tipos de datos son coherentes con el dominio de cada atributo.
- [ ] El modelo físico implementado en SQL coincide con el modelo lógico (ER).

Cada equipo debe marcar los puntos cumplidos y agregar comentarios en caso de excepciones o decisiones de diseño.
